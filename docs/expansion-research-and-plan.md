# StreamHub 확장 — 실서비스 조사 & 실현가능성 플랜

> 요청: 위치기반 교회찾기·예배신청·교단정보·CCM 음반 커머스(미리듣기/배송/오프라인매장)·결제·SMS·챗봇·AWS 라이브를 실제 운영 서비스 조사 기반으로 도입.
> 본 문서 = 웹 리서치(2026-06) 결과 + 기능별 실현가능성 + 단계별 플랜. **구현 전 우선순위 합의용.**

---

## 0. 핵심 결론 (먼저 읽기)

1. **교단별 교회 "공공데이터"는 사실상 없다.** data.go.kr·LOCALDATA·Kakao·Naver 어디에도 **교단(감리교/장로교…) 구분 필드와 교회 좌표를 함께 제공하는 데이터셋이 없음.** 교단 정보의 권위 출처인 교단 공식 주소록(KMC `his.kmc.or.kr`, 예장통합 `pck.or.kr/address.php`)은 **조회 UI만 제공, API·다운로드·좌표 없음.**
2. **그래서 위치기반 교회 검색은 "데이터 보유"가 아니라 "지도 API 호출"로 만든다.** → **Kakao Local 키워드검색**(`query=교회`, `radius ≤ 20km`) + 브라우저 **Geolocation**(거부 시 IP fallback). 무료·즉시 구현.
3. **교단 라벨은 두 가지로 근사**: ① 교회명 문자열 휴리스틱("○○**감리**교회"/"○○**장로**교회") ② 교회114(`ch114.kr`, 70+교단 분류 운영) 패턴을 참고한 소수 **시드 수동수집**.
4. 나머지(결제·SMS·챗봇·라이브)는 **실제 연동 가능**하나 포트폴리오에선 **샌드박스/목업이 정답**(사업자·비용 회피). 특히 **챗봇은 이미 보유한 LLM 스택으로 진짜 동작** 가능.

## 1. 교회 위치/교단 데이터 (리서치 상세)

| 출처 | 좌표 | 교단필드 | 접근 | 데모 적합 |
|---|---|---|---|---|
| 문체부 "한국 종교현황" (data.go.kr/3075609) | ✕(PDF 통계) | ✕ | 무료파일 | ✕ |
| **LOCALDATA**(localdata.go.kr) 지방인허가 | △(EPSG:5174, WGS84 변환 필요) | ✕ | 무료 OpenAPI/CSV, 매일갱신 | △ |
| 교단 주소록 KMC/PCK | ✕ | ◎(유일) | 조회 UI만(API·다운로드 없음) | ✕(자동화 불가) |
| **Kakao Local 키워드검색** | ◎(x,y) | ✕ | `KakaoAK` 무료키, radius≤20km | **◎** |
| Naver 지역검색 | △ | ✕ | 5건 제한(페이징 약함) | △(보조) |
| 교회114 ch114.kr | — | ◎(70+교단 분류 실제 운영) | 웹 UI | 참고모델 |

- **IP→위치**: 브라우저 Geolocation(정확, 권한필요) 주 + `ip-api.com`(키없음, 45req/분) fallback. 반경검색은 Kakao `radius`가 처리.
- **결론**: "내 주변 교회"는 ◎ 구현 가능. "교단별 정밀 분류"는 데이터 한계로 **휴리스틱/시드 근사**가 현실해.

## 2. 교회 서비스 벤치마크 (차용 기능)

- **교회114**(ch114.kr): 지역×교단×교회명 다중필터, 결과카드(교단·주소·전화·담임목사). → **필터 검색 UI 표준**.
- **섬김 앱**: 현위치 거리순 정렬 + 시설정보(주차/승강기/층수) + 전화·문자 문의. → **거리순 + 연락 CTA**.
- **새가족 등록 폼**(사랑의교회 표준): 인적/주소(우편번호검색)/등록부서/신앙배경/인도자/가족(최대5)/사진/개인정보동의. → **단계별 폼 + Zod 검증 + 동적 가족배열** (예배신청의 실체).
- **예배안내 테이블**(영락교회): 예배종류×시간×장소×대상.
- **온라인예배**: 한국 표준 = **YouTube 라이브 임베드**(별도 "신청" 모델 희소). → 라이브 임베드 + "예배 시작 알림 신청"이 현실적.

**Top5 차용**: ① 위치기반 내주변교회(거리순+지도마커) ② 다중필터 검색 ③ 새가족/예배 온라인 등록폼 ④ 교회상세(예배시간+길찾기) ⑤ 설교 아카이브(태그검색)+라이브 임베드.

## 3. CCM/음반 커머스 (리서치 상세)

- 실제: 갓피플몰·**두란노몰**(오프라인 직영점 다수, 배송 17,000원↑무료)·생명의말씀사(직영서점7)·멜론CCM·벅스CCM.
- **미리듣기**: 멜론/벅스 = 비로그인 1분 프리뷰. 두란노 = **IE/ActiveX 레거시**(모던 브라우저 재생불가).
- **차별화 스토리**: HTML5 `<audio>` **30초 프리뷰**로 "레거시 IE 의존 없는 즉시 미리듣기" — 데이터모델 `Album→Track(1:N)` + `previewUrl`, 단일 전역 오디오 인스턴스(한 곡만 재생).
- **오프라인 매장**: 직영점 위치 = #1 지도 기능 재활용(매장 찾기). 배송/구매관리 = 기존 굿즈/주문 도메인 확장으로 흡수 가능(CCM=굿즈 카테고리 "음반").

## 4. 결제·SMS·챗봇·라이브 (실현가능성)

| 기능 | 실제 연동 | 권장 |
|---|---|---|
| **결제** | 토스페이먼츠(가입거의불필요 `test_ck_`)◎, PayPal 샌드박스◎, 카카오페이 `TC0ONETIME`△, 네이버페이✕(사업자필수) | **토스+PayPal 샌드박스**, 멀티PG는 PortOne 경유. 카드결제=토스 테스트키 |
| **SMS** | 알리고(`testmode_yn=Y` 과금0)◎, SOLAPI(300P 무료실발송)◎, NHN/NCP△ | **알리고 또는 SOLAPI**. ⚠️발신번호 사전등록 필수, 알림톡만 템플릿승인. "커스텀 발송"=일반 SMS 자유본문 |
| **챗봇** | 자체 LLM(Claude/OpenAI)+Tool Calling◎, Dialogflow◎, 카카오 i 오픈빌더✕(사업자심사) | **자체 LLM 챗봇**(이미 스택 보유) — 상품문의/FAQ/주문조회 실연 |
| **AWS 라이브** | IVS Low-Latency◎(상시구동 과금), MediaLive✕(idle도 과금), YouTube임베드◎ | **HLS 샘플 목업 기본 + IVS Basic 단발 실연 캡처** 하이브리드 |

> 공통: 실제 결제·SMS·라이브를 24/7 켜두면 비용/사업자 이슈 → **포트폴리오는 샌드박스/테스트모드/목업**이 정답. 챗봇만 진짜로 돌릴 가치 큼.

## 5. 권장 단계별 플랜 (큐레이션 — 전부 X, 임팩트 순)

| Phase | 기능 | 데이터/기술 | 임팩트 |
|---|---|---|---|
| **C1 교회 위치/검색** | 내주변 교회(Geolocation+IP fallback) + 다중필터(지역×교단×명) + 교회상세(예배시간·전화·길찾기) | **Kakao Local API** + 시드 교회(교단 휴리스틱) | ★최고(기술+도메인) |
| **C2 예배/새가족 신청** | 단계별 등록 폼(인적/주소/신앙/가족동적배열) + 관리자 신청 목록 | RHF+Zod + 다음 우편번호 | ★높음(폼 UX) |
| **C3 CCM 음반 커머스** | 앨범/트랙 + **30초 HTML5 미리듣기** + 오프라인매장찾기(지도 재활용) + 구매=기존 주문도메인 흡수 | Album→Track, previewUrl | ★높음(차별화) |
| **C4 결제 연동** | 토스페이먼츠 + PayPal **샌드박스** 결제 플로우(주문→결제→영수증) | 테스트키 | ★중(실연동 어필) |
| **C5 챗봇** | LLM 기반 상품문의/FAQ/주문조회 챗봇(Tool Calling) | 보유 LLM 스택 | ★중(실동작) |
| **C6 SMS 알림** | 주문/예배신청 SMS 알림(테스트모드) + 관리자 커스텀 발송 | 알리고/SOLAPI 테스트 | ★중(기존 SMS도메인 확장) |
| **C7 라이브** | 교회 라이브(HLS 목업) + 설교 아카이브 | hls.js + IVS 단발캡처 | ★중(목업) |

**전제(사용자 확인 필요)**:
- Kakao Developers 키(무료) 발급 가능 여부 — C1 핵심.
- 결제/SMS는 **샌드박스/테스트모드**로 진행(실 사업자 가입 불필요) 합의.
- 라이브는 **목업 기본** 합의(실 IVS는 비용 발생).

## 6. 데이터/리스크 메모
- 교단 정밀데이터 부재 → 교회 시드는 **Kakao 검색 결과 + 교회명 휴리스틱 교단 라벨** 또는 소수 수동 시드. "공식 교단 데이터 연동"은 불가하다고 정직 표기.
- 실제 교회 상호/전화를 그대로 쓰면 PII/상호 노출 → **데모용 가공**(마스킹/가상) 권장, 또는 Kakao 결과를 런타임 표시만(저장 최소).
- 결제/SMS/라이브 전부 "테스트·데모" 배지 명시(실거래/실발송 아님).
